# Species Archetypes

## Overview
This document describes the AmP Species Archetype database generated by the TwoTimescaleResilience framework. This database classifies species into broad response-capacity archetypes using analytical metrics computed over DEB (Dynamic Energy Budget) physiological axes parameters extracted from the AmP (Add-my-Pet) library.

The database provides a reusable set of functional groupings for examples, demonstrations, and grid simulations to easily select "representative species" that show specific physiological vulnerability or resilience traits when subjected to stressors, without requiring the dynamic computation of these traits each time.

**Important Context:**
- This database is derived from actual AmP species response-capacity diagnostics.
- It does **not** create or modify species.
- It does **not** modify AmP parameters.
- Archetypes are framework-specific vulnerability/response-capacity labels, built strictly upon analytical Fmax (amplification potential), minimum collapse burden, and dominant sensitive axes under this project's formulation.
- Archetypes are strictly for selecting representative species in diagnostics and grid examples.
- They are **not** ecological value rankings, nor are they formal DEB taxonomic or physiological classes.

## Archetype Components
The generated archetypes reflect several key vulnerability metrics based on deterministic calculation logic:

### Amplification Potential (`Fmax`)
High amplification potential indicates a large maximum possible restoring-force amplification under conditions of margin depletion.
- `low_amplification_potential`
- `intermediate_amplification_potential`
- `high_amplification_potential`
- `extreme_amplification_potential`

### Collapse Threshold (`min_x_collapse`)
A low collapse threshold means relatively little background axis burden is needed to completely collapse the adaptive margin (i.e., high sensitivity to background stress). Conversely, a robust or high collapse threshold means the species has a large reserve margin relative to the parameter impact weighting (`alpha`).
- `high_sensitivity_low_collapse_threshold`
- `intermediate_collapse_threshold`
- `robust_high_collapse_threshold`
- `extreme_robust_high_collapse_threshold`

### Axis Sensitivity
Labels reflecting the most sensitive physiological process axis in DEB.
- `assimilation_sensitive`
- `maintenance_sensitive`
- `growth_sensitive`
- `reproduction_sensitive`

### Dominant Axis Loadings
Labels classifying species according to heavily skewed alpha weightings across axes.
- `strongly_assimilation_weighted`
- `strongly_maintenance_weighted`
- `strongly_growth_weighted`
- `strongly_reproduction_weighted`
- `balanced_axis_sensitivity`

### Combined Classifications
Several combined archetype labels are generated to simplify selections for more specific experimental cases:
- `high_Fmax_maintenance_sensitive`
- `high_Fmax_growth_sensitive`
- `high_Fmax_reproduction_sensitive`
- `low_threshold_maintenance_sensitive`
- `low_threshold_growth_sensitive`
- `low_threshold_reproduction_sensitive`
- `robust_low_amplification`
- `vulnerable_high_amplification_low_threshold`

## Usage
The archetype databases are stored in the `data/` directory and can be used in downstream scripts.
- `data/AmP_Species_Archetypes.csv` provides a readable table where species are grouped along with their relevant calculated parameters.
- `data/AmP_Species_Archetypes.json` provides an identical structured object array that programmatically loads into Julia efficiently with complete metadata descriptors mapping exact derivation scripts.
