# M1 — how much independent signal is in the feature set?

*Working note. After making the adaptive-margin state primary (M3) and adding the
capacity-aware absolute-margin features (M2), this checks empirically: are the
F-features redundant, and do the new capacity features survive deduplication?
Generated by `examples/feature_redundancy_check.jl` on a synthetic-but-realistic
scenario — 8 real AmP species spanning A0 (2.7 → 2.86×10⁶), a spatial
species-composition gradient, and an axis/season-varying stress field.*

## Headline numbers

`drop_redundant` at |r| ≥ 0.95 removes **29 of 40 features (≈72%)**.

| | input → kept |
| --- | --- |
| all features | 40 → 11 |
| F-derived | 11 → 3 |
| capacity / absolute-margin | 4 → 1 |

- `cor(mean_F, mean_Q) = 0.67` — F is **not** redundant with load.
- `cor(mean_log_absolute_margin_remaining, mean_relative_margin) = 0.74` — the
  capacity feature is **not** redundant with relative margin.
- `mean_relative_margin_remaining` is dropped (r = −1.000 with `mean_Q`) — it *is* −Q.

## What it means

1. **The feature set is ~70% redundant.** The distribution summaries
   (`median/p90/p95/max/sd`) of `Q` and `F` are near-perfect duplicates of their
   means, and relative margin is literally `1 − Q`. A lean, non-redundant set is
   ~11 features. Turning on `drop_redundant` (or curating) is worth it.

2. **M2 is validated: the capacity-aware absolute margin survives dedup.** It
   carries signal the relative margin does not (r ≈ 0.74), so adding it genuinely
   restored a clustering dimension that the `F`-ratio had cancelled. The four
   capacity variants collapse onto one — keep `mean_log_absolute_margin_remaining`;
   the depleted/baseline/min variants are redundant with it.

3. **`F` is over-represented, not pure dead weight.** One `F` summary survives
   (`cor(mean_F, mean_Q) = 0.67`, so it carries the amplification/`g` axis beyond
   load) but the six `F`-distribution features collapse to it. The catch: the axis
   `F` carries is `g`, which the life-history check showed is not a validated
   vulnerability axis. So **keep a single `F` summary, drop the rest**, and treat
   the substantive multi-dimensional signal as **load (`Q`) + capacity (absolute
   margin) + mode-of-action composition (`E_a`, entropy, dominant axis) + timing.**

## Caveat

The *exact* cross-correlations are scenario-specific — the panel is sorted by `A0`
and the composition gradient is aligned with capacity, which inflates e.g.
`cor(baseline_capacity, F) = −0.91` and `cor(baseline_capacity, relative_margin) =
0.72`. Those particular numbers would differ on a real raster. The **robust**
findings are the redundancy structure (distribution summaries duplicate means) and
that the capacity feature survives dedup carrying independent signal.

## Practical upshot

The clustering layer already contains the right multi-dimensional signal; it is
just heavily duplicated and `F`-heavy. The margin-first cleanup is: (a) deduplicate
(one summary per family), (b) keep the capacity-aware absolute-margin feature, (c)
demote `F` to a single derived summary — leaving load, capacity, axis composition,
and timing as the independent vulnerability dimensions.
